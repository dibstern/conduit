# Task 5: TLS Certificate Loading as a Layer -- Audit Report

**Summary:** The plan has a behavioral bug (missing `hostExplicit` guard), a missing field in the service interface (`caCertPem`), a missing import for `ensureCerts`, and insufficient test specification. Layer composition order is not explicitly stated but is inferrable from the design doc.

## Findings

| 1 | State Issues | Amend Plan | **Missing `hostExplicit` guard when updating host to `0.0.0.0`.** The existing imperative code at `daemon-main.ts:1062` checks `!hostExplicit` before changing host to `0.0.0.0`. The plan's `TlsCertLive` code checks `c.host === "127.0.0.1"` instead. This is not equivalent -- a user who explicitly passes `--host 127.0.0.1` would have their host overridden to `0.0.0.0` by the plan's code, which is wrong. The `DaemonRuntimeConfig` interface (Task 1) has no `hostExplicit` field. | Plan Task 5 code, line `c.host === "127.0.0.1" ? Option (a) preserves existing semantics. |

| 4 | Implicit Assumptions | Amend Plan | **`ensureCerts` signature takes `TlsOptions` (optional), not `{ configDir }`.** The plan calls `ensureCerts({ configDir })` which is correct -- `TlsOptions.configDir` is optional.
