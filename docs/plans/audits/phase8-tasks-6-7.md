# Phase 8 Audit: Tasks 6-7 (ProjectRegistry + InstanceManager)

## Summary

Both tasks describe converting imperative services to Effect Layers. However, the plan text is significantly out of date: the actual source files already contain fully implemented Effect-native versions of both services. Task 6 and Task 7 propose creating code that already exists. The plan also proposes adding DaemonEvent variants that are unnecessary (the existing events already cover the use cases). Several event name mismatches and missing methods need attention.

## Key Recommendation

Tasks 6 and 7 need substantial rewriting. The core implementation work is already done. The tasks should be refocused on:
1. Verifying coverage of all imperative consumers
2. Adding missing functions (broadcastToAll, waitForRelay, evictOldestSessions, replaceRelay)
3. Wiring existing Layer factories into `makeDaemonLive`
4. Deleting imperative code from daemon-main.ts
5. Bridging or replacing `InstanceMgmtTag` consumers
